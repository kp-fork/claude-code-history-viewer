# 0000 — Record architecture decisions

- **Status**: accepted
- **Date**: 2026-07-12

We will record architecture and cross-cutting decisions for this project as
numbered ADRs in `docs/adr/`, created with `/harness-kit:adr <title>`. A decision
qualifies when it is non-obvious or hard to reverse — a module boundary, a
trade-off, a technology choice (e.g. how a new provider is abstracted, or the
Tauri/Axum command-parity contract). Each ADR states the context, the decision,
and its consequences, and is never edited after acceptance — superseding
decisions get a new ADR that links back.
